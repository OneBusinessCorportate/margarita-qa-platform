# Ответы на отзыв Маргариты (июнь 2026)

Ниже — что сделано по каждому пункту отзыва и по комментариям руководителя.

## Отзыв Маргариты

1. **Горизонтальная прокрутка.** Таблица оценки переведена в «плотный» режим
   (`dense`): уменьшены ширины колонок (№/чат — 190–230px, статусы рассылок —
   84px, отступы), таблица растянута на всю ширину. Теперь основная информация и
   кнопка «Оценить» помещаются на один экран без прокрутки вправо.
   _Файлы: `src/components/ScoringPanel.tsx`._

2. **Медленная загрузка чатов / иногда открываются некорректно.** По умолчанию
   используется быстрый веб-клиент Telegram «K», все чаты открываются в одной
   переиспользуемой вкладке (без перезагрузки). Полностью проблема на стороне
   Telegram Web и от платформы не зависит, но клиент «K» заметно стабильнее «A».

3. **Отдельная оценка бухгалтера и менеджера в одном чате.** В строке чата уже
   была кнопка «+ Менеджер / + Юрист»; теперь менеджер и юрист оцениваются по
   **тем же критериям**, что и бухгалтер (Точность + SLA → 0–100), а не по
   штрафной модели. Их оценки попадают в отчёт в отдельных блоках «Менеджеры /
   Юристы — сервис в чатах».
   _Файлы: `ScoringPanel.tsx` (RoleQaRow), `report.ts` (`perPersonScores`),
   `repo.ts`, `ReportView.tsx`._

4. **Доработка интерфейса полей.** Вместе с п.1: компактные селекты, выровненные
   строки AI/Вы, бейджи статусов чата на одной строке.

5. **Поле «Долги» — вернуть сумму задолженности.** В карточке чата снова виден
   бейдж `💰 долг N ֏` (из системы долгов OneBusiness). Сумма берётся из
   `mqa_chats.debts`.
   _Файлы: `ScoringPanel.tsx`, `chat-list.ts` (`debtAmountLabel`)._

6. **Новые чаты / чаты без HVHH.** Новые чаты (создан в течение 3 дней) помечаются
   бейджем `🆕 новый` и попадают в «Активные за день» в день создания — даже до
   первой активности в ленте. Несовпадение «у Маргариты 913 / у бота 770» — это
   рассинхрон таблицы и Supabase (см. ваш Apps Script сверки); чаты без рабочей
   Telegram-ссылки бот не отслеживает. Для полного покрытия нужно, чтобы у чата
   была корректная Telegram-ссылка.

7. **Сообщения после 19:00 — к какому дню относятся.** По решению руководителя:
   после вашей проверки чата любое новое сообщение относится к **следующему**
   проверяемому периоду. Реализовано как бейдж `🔄 новое после оценки` на уже
   оценённом чате (см. п.9) — он подсказывает перепроверить чат на следующий день
   и не потерять его.

8. **Чаты «вернусь через 2 дня».** Решается через раздел **Задачи**: создаёте
   задачу с дедлайном (поле «Срок»). Когда срок подходит — задача подсвечивается
   красным («⏰ срок подошёл») и показывается счётчик «Подошёл срок». Чат при этом
   оценивается по обычным критериям.
   _Файлы: `TasksPanel.tsx`, `scoring.ts` (`isTaskDue`)._

9. **Новое сообщение после оценки.** Чат остаётся с прежней оценкой, но получает
   бейдж `🔄 новое после оценки` (сравнение времени последнего сообщения и времени
   вашей оценки), чтобы вы перепроверили его в следующий день.
   _Файлы: `chat-list.ts` (`hasNewMessageAfterEval`), `ScoringPanel.tsx`._

10. **Автодобавление критичных чатов в «Нарушения».** На странице «Нарушения»
    появилась кнопка **«➕ Импортировать критичные чаты»** за выбранную дату: все
    чаты с оценкой «Критично» автоматически попадают в журнал (важность
    «Критичное», причина — из оценки). Повторный запуск не создаёт дублей.
    _Файлы: `ViolationsPanel.tsx`, `api/violations/import-critical`, `repo.ts`._

11. **Раздел «Сообщения» — критичные чаты по бухгалтерам.** Добавлен блок
    **«Сообщения бухгалтерам»**: отдельная карточка с готовым текстом для каждого
    бухгалтера, у кого есть критичные чаты, низкие оценки или чаты без ответа —
    каждую можно скопировать и сразу отправить. Раньше была только одна общая
    аналитика, поэтому критичные чаты (напр. у Оли) некуда было скопировать.
    _Файлы: `templates.ts` (`buildAccountantMessage`, `accountantsToMessage`),
    `app/(app)/messages/page.tsx`._

## Комментарии руководителя

- **п.7 / п.9** — реализовано как описано выше (бейдж «новое после оценки»).
- **п.8** — через «Задачи» (дедлайн + подсветка срока).
- **п.10** — реализовано (кнопка импорта критичных чатов).
- **Список «сделано в этом периоде или нет» (как во вкладке Задачи 26)** —
  статусы рассылок (налоги/ЗП/первичка/долги) по каждому клиенту уже видны и
  автозаполняются в таблице оценки. Отдельный сводный per-client чек-лист периода
  можно вынести в отдельную вкладку следующим шагом, если нужно именно в формате
  Задачи 26.
- **Незакрываемые / повторяющиеся задачи** — у задачи есть флаг «🔁 Повтор».
  Повторяющаяся задача закрывается **только** после того, как бухгалтер её выполнил
  И QA подтвердил («QA подтвердил»). До этого она остаётся открытой в каждом
  периоде. Фильтр «Только открытые» включён по умолчанию.
  _Файлы: `TasksPanel.tsx`, `scoring.ts` (`isTaskClosed`), миграция
  `db/migrations/20260619_mqa_tasks_recurring_qa_confirm.sql`._
